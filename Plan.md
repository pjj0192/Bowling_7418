# Plan

볼링 점수 계산기(`Game` 클래스)를 RED-GREEN-REVIEW 사이클로 구현한다.
각 항목은 하나의 사이클(RED 커밋 -> GREEN 구현 -> REVIEW 승인 후 커밋)에 대응한다.

## Cycle 1 (완료)

- **테스트**: 모든 투구가 거터볼(0핀)이면 총점은 0점이다.
- **왜**: `roll`/`score`의 가장 단순한 골격을 세우는 케이스. 아직 스페어/스트라이크 로직 없이도 통과 가능한 최소 시작점.
- **결과**: `roll()`이 핀 수를 기록하고 `score()`가 단순 합산하는 구현으로 통과. 리뷰 승인 후 커밋됨.

## Cycle 2 (진행 중)

- **테스트**: 한 번의 스페어(예: 5+5) 뒤의 다음 투구(예: 3)가 해당 프레임의 보너스로 더해진다.
- **왜**: 현재 구현(단순 합산)은 보너스 개념이 없어 이 테스트에서 실패해야 하며, 스페어 감지 및 보너스 가산 로직을 도입하게 만드는 케이스.
- **투구 시퀀스**: 5, 5(스페어), 3, 이후 17번의 거터볼(총 20투구)
- **기대 점수**: 프레임1 = 5+5+3(보너스) = 13, 프레임2 = 3+0 = 3, 나머지 0 → 총 16점
- **예상 RED**: 현재 구현은 `sum(rolls)` = 13이 나오므로 `assert 13 == 16`으로 실패해야 한다.

## 다음 후보 (아직 미착수)

- 스트라이크: 첫 투구로 10핀 → 다음 두 번의 투구가 보너스로 더해진다.
- 퍼펙트 게임: 12번 연속 스트라이크 → 300점.
- 10번째 프레임에서 스페어/스트라이크 시 추가 투구 처리.
